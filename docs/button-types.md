# ToolHub 按钮类型

## 用户可创建类型

按钮编辑器只提供以下四种动作类型：

| 类型 | 配置 | 行为 |
|---|---|---|
| `shell` | 命令、Root 开关 | 通过 Shell 广播桥执行命令 |
| `app` | 包名、启动用户 ID | 启动主用户、分身或其他资料用户中的应用 |
| `broadcast` | Action、Extras | 发送自定义广播 |
| `shortcut` | 包名、快捷方式 ID、`intentUri` | 启动应用提供的结构化快捷方式 |

## App 选择器

App 类型保留手动填写包名和用户 ID，同时提供已安装应用选择器：

- 支持按应用名称、包名和用户 ID 搜索。
- 可分别显示或隐藏用户应用、系统应用、分身/其他用户应用。
- 使用 `LauncherApps` 按用户空间枚举可启动 Activity。
- 选中应用后回填包名和启动用户 ID；标题为空时同时回填应用名称。
- 分身启动失败时不会回退启动同包名的主用户应用。

## 已移除类型

- `open_settings` 不再作为用户可创建类型。设置入口由主面板工具栏提供；运行时分支仅保留给默认配置和配置损坏时的救援入口。
- 独立的 `intent` 类型已移除。旧配置在按钮管理保存时迁移为执行 `am start` 的 Shell 按钮。

快捷方式内部的 `intentUri` 是结构化快捷方式启动数据，不属于已移除的独立 `intent` 按钮类型。
